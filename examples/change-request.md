# Change Request

Type: minor_feature
Title: Add a small bounded feature
Why: Describe why this iteration matters now.
User-visible impact: Explain what users will notice.
Affected areas: List pages, flows, or services touched.
Urgency: medium

Notes:
- Use `bugfix` for regressions or incorrect behavior.
- Use `minor_feature` for bounded iteration that does not require major replanning.
- Use `major_feature` when scope or architecture meaningfully expands.
- Use `replan` when the current product plan itself needs revision before coding.
